## 通用设置

这些是影响您整体 SillyTavern 体验的核心设置。

### 界面语言

SillyTavern 的用户界面支持多种语言。语言选择器提供以下选项：
* **默认**：如果可用，使用您的系统语言
* **English**：无论系统设置如何，强制使用英文界面
* 其他语言可通过下拉菜单选择

注意：此设置仅影响用户界面文本。如需 AI 对话翻译，请使用[聊天翻译](../../extensions/Translation.md)扩展。

### 软件版本

您当前的 SillyTavern 版本显示在右上角。此信息对以下情况至关重要：
* 排查问题
* 确保与扩展的兼容性
* 确认是否有可用更新

要将 SillyTavern 更新到最新版本，请参阅[更新](/Installation/Updating)文档。

### 账户管理 {#account-management}

控制您的 SillyTavern 用户账户、备份设置和用户数据，并在[多用户模式](/Administration/multi-user.md)下管理用户角色和权限。

#### <i class="fa-fw fa-solid fa-user-shield"></i> 账户

在账户对话框中，您可以查看和编辑个人资料信息、更改密码以及管理账户设置。

**个人资料信息**

* 显示名称（可通过铅笔图标编辑）
* 用户头像（也可使用[人物设定](/Usage/personas.md)更改）
* 账户标识
* 用户角色
* 账户创建日期
* 密码状态（锁定/解锁图标表示是否受保护）

**账户操作**

* **设置快照**：创建、管理和恢复用户设置备份
* **下载备份**：导出所有用户数据的完整备份
* **更改密码**：更新账户安全凭据

**危险区域**

需谨慎使用的关键账户操作：
* **重置设置**：将所有设置恢复为出厂默认值
* **重置一切**：完全清除账户并恢复出厂设置

!!! 适用于：[多用户模式](/Administration/multi-user.md)

多账户功能需要在 config.yaml 中将 `enableUserAccounts` 设置为 true。
!!!

选择**管理用户**以查看和管理现有用户账户。

- 自定义头像管理（上传/移除）

##### 管理操作

- <i class="fa-fw fa-solid fa-download"></i> 下载用户数据备份

* 显示名称*（例如"John Snow"）

创建新用户会自动在 /data/ 目录下以用户标识为名创建子文件夹。

## 角色处理

* **角色列表副标题**：选择在[<i class="fa-fw fa-solid fa-address-card" title="角色图标"></i> 角色](/Usage/Characters/characterdesign.md)列表中角色名下方显示的附加信息：
    - 角色版本
    - 创建者
* **导入卡片标签**：控制导入角色卡时标签的处理方式：
    - 询问——每次导入时显示对话框
    - 无——不导入任何标签
    - 全部——导入所有标签
    - 已有——仅导入已存在的标签
* **高级角色搜索**：启用后，使用模糊匹配并搜索所有角色数据字段，而不仅仅是名称。
* **优先使用角色提示词**：如果启用，则在可用时使用角色卡的系统提示词覆盖。
* **优先使用角色指令**：如果启用，则在可用时使用角色卡的历史后指令覆盖。
* **不缩放头像**：防止对导入的角色图像进行裁剪/缩放。禁用时，图像将被缩放至 512x768。
* **显示头像文件名**：在角色列表中显示角色头像的实际文件名。
* **剧透免疫模式**：在编辑器面板中，将角色定义隐藏在剧透按钮后面。

## 杂项 {#miscellaneous}

* **重新加载聊天**：重新加载并重绘当前聊天。
* **[调试菜单](#debug-menu)**：访问调试选项。
* **平滑流式传输**：通过逐字显示文本来平滑生成的流式传输内容，包含速度控制滑块。若要从平滑流式传输中排除推理块，请启用"排除'思考中...'"选项。
* **流式淡入**：对流式传输的文本应用淡入效果。可与平滑流式传输配合使用或单独使用。
* **[消息提示音](uicustomization.md#message-sound)**：消息生成完成时播放提示音。
    - **仅后台提示音**：仅在浏览器标签页未聚焦时播放提示音。
* **宽松的 API URL**：降低对 API URL 格式的要求。
* **知识库导入对话框**：在导入带有内嵌知识库的角色时，显示世界信息/知识库的导入对话框。
* **自动选择输入文本**：单击某些输入字段时自动选中其中的文本。
* **Markdown 快捷键**：启用 Markdown 格式的键盘快捷键。
* **恢复用户输入**：页面刷新时保留未保存的用户输入。
* **移动 UI**：允许通过拖拽重新定位 UI 元素（仅限 PC）。
    - <i class="fa-solid fa-recycle" title="重置图标"></i> **重置**按钮，用于恢复默认位置
    - 用于保存/加载 UI 布局的预设系统

## 聊天/消息处理

### 消息显示设置

控制消息在聊天界面中的加载和显示方式。这些设置会影响整体聊天体验和性能。
* **加载消息数量**：分页前加载的聊天历史消息数（0 = 全部）
* **流式传输帧率**：流式传输文本的更新速度（5-100 FPS）
* **示例消息行为**：
    - 逐步推出
    - 始终包含示例
    - 从不包含示例
* **图像滑动行为**（控制图库样式中图像的滑动操作）：
    - 生成新图像：允许使用[图像生成](/extensions/Stable-Diffusion.md)扩展生成新图像
    - 循环滚动：在现有图像间循环，到达末尾时从头开始

### 输入与响应控制

决定消息发送方式以及 AI 如何继续响应的设置。
* **回车发送**：在禁用、自动（PC）或启用之间选择
* **"发送"继续**：使用发送按钮继续 AI 的响应
* **快速"继续"按钮**：显示用于延长 AI 最后一条消息的按钮
* **快速"模拟"按钮**：显示用于单条消息角色模拟的按钮
* **滑动**：显示用于备选 AI 响应的箭头按钮（PC 和移动端）
* **手势**：启用滑动手势进行生成（仅限移动端）

### 自动管理

帮助管理聊天流程和内容的自动化功能。
* **自动加载最近聊天**：启动时自动加载最近的聊天
* **自动滚动聊天**：自动滚动到最新消息
* **自动保存消息编辑**：无需确认即保存消息编辑
* **确认消息删除**：删除消息前提示确认
* **自动修复 Markdown**：自动修正 Markdown 格式

#### 自动滑动

根据可配置的条件自动拒绝并重新生成 AI 消息。
* **启用自动滑动**：自动滑动功能的主开关
* **生成消息的最小长度**：若消息长度短于此值，则触发自动滑动
* **黑名单词语**：可触发自动滑动的词语列表，用逗号分隔
* **触发滑动所需的黑名单词语数量**：触发自动滑动所需检测到的最少黑名单词语数

#### 自动继续

如果模型在达到特定长度之前停止，则自动继续响应。

这让您的 AI 可以分多段撰写长篇响应，从而在[响应长度设置](/Usage/Common-Settings.md#response-tokens)较短的情况下仍能获得较长的回复。

这不会让 AI 写出超出其本意的内容。让 AI 继续它认为已"完成"的消息通常不起作用。请参阅[如何让 AI 写更多内容？](/Usage/faq.md#how-to-make-the-ai-write-more)以获取其他建议。

* **启用自动继续**：自动继续功能的主开关
* **允许聊天补全 API**：为聊天补全 API 端点启用自动继续功能
* **目标长度（词元数）**：期望的消息长度（以词元为单位）——若消息短于此值则触发继续（0-1024）

### 消息格式与显示

控制消息格式及显示内容。
* **禁止外部媒体**：屏蔽来自外部域名的嵌入媒体
* **在响应中显示 {\{char}}:**：如果生成了角色名前缀，则在响应中保留
* **在响应中显示 {\{user}}:**：如果生成了用户名前缀，则在响应中保留
* **实验性宏引擎**：启用支持嵌套的高级宏处理
* **在响应中显示标签**：允许响应中的（部分）HTML 标签以 HTML 形式显示
* **放宽群聊消息修剪**：允许 AI 在群聊中为其他角色发言，而不是停止响应生成
* **显示群聊队列**：在群聊的角色列表中显示响应顺序
* **固定问候消息样式**：始终渲染问候语中的样式标签，即使由于懒加载该消息已被卸载。

### 提示词检查与调试

* **将提示词记录到控制台**：将提示词输出到浏览器控制台
* **请求词元概率**：从 API 请求 AI 响应的词元概率。在可用的情况下，可在 <i class="fa-solid fa-bars" title="菜单图标"></i> [词元概率](../../Usage/Chatting/index.md#token-probabilities-panel)中查看。

### 自动补全

- 自动隐藏详情
- 匹配样式（以...开头/包含/模糊）
- 视觉样式（主题/深色/浅色）
- 键盘选择选项
- 字体缩放
- 宽度控制

## 清理菜单

清理菜单提供了一个数据维护工具，帮助您识别并删除 SillyTavern 安装中不必要的文件。此功能有助于保持数据目录的整洁，并可释放大量磁盘空间。

!!! warning "重要警告"
清理工具将永久删除文件。**此操作不可撤销！**

手动上传到 `/data/user/files/` 和 `/data/user/images/` 目录的文件，如果未与聊天消息或数据库条目关联，将被删除。

如有疑问，请在使用清理菜单前备份您的数据。
!!!

### 如何使用清理功能

1. 单击**杂项**部分下的**清理**按钮
2. 单击**扫描**以分析您的安装。根据数据目录的大小，这可能需要一些时间
3. 查看找到的文件类别
4. 使用**查看**在删除前预览文件内容
5. 使用**下载**在删除前保存文件
6. 根据需要删除单个文件或整个类别

### 清理类别

清理工具会将松散文件扫描到以下类别中：

#### 文件

* **查找内容**：未与聊天消息或数据库条目关联的文件
* **位置**：`/data/<用户标识>/user/files/`
* **风险**：⚠️ **将删除未在聊天中引用的手动上传文件**
* **何时清理**：如果您不需要未引用的文件，则可安全删除

#### 图像

* **查找内容**：未与聊天消息关联的图像
* **位置**：`/data/<用户标识>/user/images/`
* **风险**：⚠️ **将删除未在聊天中引用的手动上传图像**
* **何时清理**：如果您不需要未引用的图像，则可安全删除

#### 聊天记录

* **查找内容**：与已删除角色关联的聊天文件
* **位置**：`data/<用户标识>/chats/`
* **风险**：⚠️ **孤立的聊天记录将永久丢失**
* **何时清理**：如果您已故意删除角色且不再需要其聊天历史，则可安全删除

#### 群聊记录

* **查找内容**：与已删除群组关联的聊天文件
* **位置**：`data/<用户标识>/group chats/`
* **风险**：⚠️ **孤立的群聊记录将永久丢失**
* **何时清理**：如果您已故意删除群组且不再需要其聊天历史，则可安全删除

#### 头像缩略图

* **查找内容**：缺失或已删除角色的头像缩略图
* **位置**：`data/<用户标识>/thumbnails/avatar`
* **风险**：✅ **可安全删除**——缩略图在需要时会自动重新生成
* **何时清理**：随时可安全清理，有助于释放空间

#### 背景缩略图

* **查找内容**：缺失或已删除背景的缩略图
* **位置**：`data/<用户标识>/thumbnails/bg`
* **风险**：✅ **可安全删除**——缩略图在需要时会自动重新生成
* **何时清理**：随时可安全清理，有助于释放空间

#### 聊天备份

* **查找内容**：自动生成的聊天备份
* **位置**：`data/<用户标识>/backups/chat_*`
* **风险**：⚠️ **备份文件将永久丢失**
* **何时清理**：建议保留最近的备份，但较旧的备份可安全删除

#### 设置备份

* **查找内容**：自动生成的设置备份
* **位置**：`data/<用户标识>/backups/settings_*`
* **风险**：⚠️ **设置备份文件将永久丢失**
* **何时清理**：建议保留最近的备份，但较旧的备份可安全删除

## 调试菜单 {#debug-menu}

!!!warning 这些功能仅供高级用户使用。

除非您完全了解其后果，否则请勿使用。
!!!

调试菜单提供了用于故障排查、维护和开发目的的功能。这些功能应谨慎使用，因为它们可能对您的 SillyTavern 安装产生重大影响。

由于扩展可以添加调试功能，因此可用选项将根据您已安装的扩展而有所不同。

### 翻译与语言区域功能
* **获取缺失翻译**：分析当前语言区域（若选择英语则分析所有语言区域）中缺失的翻译，并将结果输出到浏览器控制台
* **应用语言区域**：通过重新应用所选语言区域来强制刷新当前语言设置
### 缓存与存储管理
* **清除 WebSearch 缓存**：从本地缓存中删除所有已存储的搜索结果
* **清除所有向量索引**：完全删除所有来源中存储的所有向量
* **重置词元缓存**：清除已存储的词元计数，强制对所有聊天进行完整的重新词元化
* **删除逐项提示词**：从本地存储中删除所有逐项提示词
### 数据与统计
* **刷新统计文件**：使用现有聊天数据重建统计文件
* **补填词元计数器**：重新计算当前聊天中所有消息的词元数
    - 在切换使用不同词元化器的模型时非常有用
    - 完成后触发聊天重新加载
    - 仅为视觉变化，不修改聊天内容
### API 与扩展测试
* **更改 Mancer 基础 URL**：修改 Mancer API 服务器的基础 URL
* **测试 WebSearch 扩展**：使用当前设置执行测试搜索
* **发送生成请求**：使用当前选定的 API 测试文本生成
### 系统与调试工具
* **强制引导流程**：重新启动引导流程
* **切换事件追踪**：启用/禁用用于调试的事件跟踪
* **复制 ST 配置**：[开发中] 将系统配置数据复制到剪贴板，用于提交错误报告

每个功能均可使用其描述下方的"执行"按钮来运行。在使用这些工具之前，请考虑备份您的数据，因为某些操作无法撤销。
