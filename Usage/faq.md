# 常见问题解答

## 解释一下 SillyTavern 是什么

现代 AI 语言模型（如 ChatGPT）已经变得非常强大，其中一些现在可以令人信服地模拟你创建的角色，你可以与之聊天、一起写小说等。例如，你可以告诉 AI 假装自己是来自中世纪日本的围棋教练 Jubei，它就会相应地行动和回应。你可以与 Jubei 进行长时间的聊天，一起去酒馆，决定与武士打架，任何你能想象到的事情，AI 都会配合并围绕这些内容进行写作/反应，充当你的陪衬和地下城主。你的想象力就是极限。你可以告诉 AI 假装它是神奇女侠。你还可以指定一个场景（"神奇女侠和我正在抢劫银行"），一种写作风格（"神奇女侠说黑人英语"），或者你能想到的任何其他内容。

SillyTavern 是一个促进这些用途的应用：

* 它是一个处理与 AI 语言模型通信的用户界面。
* 它让你可以创建新的角色卡（提示词），并轻松在它们之间切换。
* 它让你可以导入其他人创建的角色。
* 它会保存你与角色的聊天历史记录，允许你随时恢复、开始新的聊天、查看旧聊天等。
* 在后台，它会为你做必要的准备工作来准备 AI 提示词。具体来说，它会发送一个系统提示（AI 的指令），引导 AI 遵循某些规则以提高响应准确性。

## 给我介绍一下 AI 模型选项

SillyTavern 可以与两种类型的 AI 交互：

1. [Web 服务](/Usage/API_Connections/openai.md)（基于云的，通常是付费的、专有的、封闭的）
2. [自托管](/Usage/API_Connections/self-hosted.md)（本地的、免费的、开源的）

### 付费 Web 服务 AI

付费 Web 模型是黑箱。你向公司付费使用他们的 AI 服务。你在 SillyTavern 中输入你的账户信息，它将连接到你的提供商代表你使用 AI。

优点：

* 非常容易上手。
* 最高质量的 AI 写作。

缺点：

* 使用需要花钱。
* 所有内容都记录在他们的服务器上。隐私问题。
* 它们通常受到审查，会拒绝与你聊天某些主题。

### 自托管 AI

自托管模型是你可以在 PC 上运行的免费模型，但需要一台强大的 PC 和更多的设置工作。

优点：

* 一旦设置好，即使没有互联网访问也可以免费使用。
* 完全隐私。你写的所有内容都保留在你自己的 PC 上。
* 有各种各样的模型。作为一项社区驱动的技术，你可以找到适合你想要的某些任务或行为的模型。

缺点：

* 它们不如 <abbr title="State of the art">SOTA</abbr> 模型（即，它们编写的对话更差，创意更少等）。
* 运行本地模型需要至少 6GB VRAM 的 GPU。

如果你有兴趣使用这些，请参考这里的专用指南：[如何使用自托管模型](/Usage/API_Connections/self-hosted.md)。

## 我可以在手机或平板电脑上使用 SillyTavern 吗？

iPhone 和 iPad 无法运行整个 SillyTavern 应用，但由于它只是一个 Web 界面，你可以在家庭 Wi-Fi 上的另一台计算机上运行它，然后在移动浏览器中访问它。有关更多信息，请参阅[远程连接](/Administration/remote-connections.md)。

对于 Android 用户，除了上述方法外，你还可以使用 Termux 应用直接在手机上运行整个 SillyTavern，无需 PC。请参阅[安装（Android）](/Installation/Android.md)。（注意：Termux 安装不受官方支持，我们不能保证它能正常工作。）

## 我尝试导入 PNG 角色卡，但收到错误提示说它无效。为什么？

两种可能性：

1. 该卡没有嵌入定义，只是一个普通的图像文件。某些程序或文件管理器在保存时会从卡中剥离嵌入的定义。确保你使用的是分享者发布的原始 PNG 文件。
2. PNG 文件实际上是一个带有 `.png` 文件名的 WEBP 文件。你可以尝试在导入前将卡重命名为 `.webp`，或者寻找图像的正确 PNG 版本。

## 我如何创建自己的 AI 角色？

1. 点击角色管理按钮
2. 点击创建新角色
3. 在角色名称下，给一个名字，比如 Amanda
4. 可选地，点击选择头像按钮为这个角色选择一个图像肖像
5. 在描述下，描述角色，并包含你认为与聊天相关的任何信息。例如：```Amanda 是一名在间隔年旅行的学生。她身高 6 英尺，是一名排球运动员。她有运动员的身材。她有一头棕色长发。她热爱维多利亚时代的英格兰，喜欢观看与那个时期相关的电视和阅读小说。```
例如，如果你想让 Amanda 友好，那么你会添加：```Amanda 非常开朗和外向。```
6. 在第一条消息下，写下当你开始新聊天时角色的问候语。例如：```*Amanda 向你挥手* 嘿！你也是背包客吗？```
7. 点击创建角色按钮

你现在有一个可以聊天的基本角色。从角色列表中选择 Amanda，新的聊天就会开始。

请注意，你可以使用描述和/或第一条消息来创建更具体的场景，和/或在描述中包含你自己。例如：

```txt
描述：
Amanda 是一名在间隔年旅行的学生。她身高 6 英尺，是一名排球运动员。她有运动员的身材。她有一头棕色长发。她热爱维多利亚时代的英格兰，喜欢观看与那个时期相关的电视和阅读小说。她一直保守着一个沉重压在她心头的秘密。她在等待合适的人来倾诉，但这可能会导致一场与强大秘密社团的猫鼠游戏。她最近到达了加尔各答。

你是 Rajesh Nahasmapetilon，一位世界著名的印度排球巨星。你在加尔各答散步。Amanda 发现了你，兴奋地尖叫起来。

第一条消息：
*Amanda 跑向你，笑容满面。* Rajesh！我真不敢相信！我是你的超级粉丝。我的卧室里有你的海报。
```

你包含的任何相关信息都可以使用。使用效果取决于 AI 模型的能力水平。

注意：一旦角色创建完成，你可以回去编辑任何这些信息，除了名字。

## 我的 API 密钥存储在哪里？为什么我看不到它们？

SillyTavern 将你的 API 密钥保存到用户数据目录中的 `secrets.json` 文件（`/data/default-user/secrets.json` 是默认路径）。

默认情况下，在你保存并刷新页面后，API 密钥将不会从界面中可见。

为了启用查看你的密钥：

1. 在 `config.yaml` 文件中将 `allowKeysExposure` 的值设置为 `true`。
2. 重启 SillyTavern 服务器。
3. 点击 API 连接面板右下角的 "查看隐藏的 API 密钥" 链接。

## 性能提示

### 为什么 UI 这么慢/卡顿？

* 尝试在用户设置面板上启用无模糊效果（快速 UI）模式。
* 在 UI 主题设置中启用减少动效以删除装饰性动画。
* 确保你的浏览器正在使用硬件加速。
* 如果使用响应流式传输，请将流式传输 FPS 设置为较低的值（推荐 10-15 FPS）。

### 我遇到输入延迟。我能做什么？

性能下降，特别是输入延迟，最常归因于浏览器扩展。已知有问题的扩展包括：

* iCloud Password Manager
* DeepL Translation
* 基于 AI 的语法纠正工具
* 各种广告拦截扩展

如果你遇到性能问题并且无法确定原因，或怀疑 SillyTavern 本身存在问题，请：

1. [记录性能配置文件](https://developer.chrome.com/docs/devtools/performance/reference)
2. 将配置文件导出为 JSON 文件
3. 将其提交给开发团队进行分析

我们建议首先测试禁用所有浏览器扩展和第三方 SillyTavern 扩展，以隔离性能下降的来源。

### 当我导入大量角色时，应用变慢了。为什么？

不幸的是，SillyTavern 并不是为处理庞大的角色库而设计的。你拥有的越多，加载角色列表所需的时间就越长。证据数据表明，当你拥有超过 1000 个角色时，性能下降开始变得明显。

但是，你可以采取一些措施来缓解这个问题：

**1. 使用延迟加载。**

在 `config.yaml` 文件中将 `performance.lazyLoadCharacters` 的值设置为 true 来启用角色的延迟加载。下次服务器重启后，角色列表将仅加载你与之交互的角色的完整数据。请注意，如果某些第三方扩展未更新以支持此设置，则可能无法正常工作（请联系扩展开发者以获取更多信息）。

**2. 使用内存缓存。**

如果你有一些空闲的 RAM，请增加内存缓存容量。这将允许服务器在内存中保留更多角色，减少加载它们所需的时间。你可以通过在 `config.yaml` 文件中将 `performance.memoryCacheCapacity` 的值调整为更高的数字来实现。默认值为 `100mb`。大致经验法则：对于你拥有的每 3000 个角色，将值增加 100mb。

**限制：**

1. 启用延迟加载后，高级（模糊）角色搜索将无法工作。仅搜索角色名称。
2. 由于可用内存有限，Android 设备上禁用了内存缓存。

## 如何让 AI 写得更多？

有时候，当你希望 AI 更详细时，它只会回复一句话。
这通常是本地运行模型的问题。

如果你只是想让机器人从其最近回复的结尾继续写作，你可以通过在输入栏中不输入任何内容并点击发送来发送空用户消息。这将强制机器人继续故事。

修复此问题的策略：

* 增加 `响应长度` 设置的值
* 为角色设计一个好的 `第一条消息`，展示他们以长篇大论的方式说话。当给出关于你期望的写作风格的指导时，AI 模型可以大大改善。
* 在角色的描述框中添加短语，例如 "喜欢说很多话" 或 "非常啰嗦的说话者"
* 对你的 `作者注释` 或 `历史后指令提示` 做同样的事情
* 作为最后的手段，你可以尝试打开 `自动继续`（在用户设置面板中），但这会使响应变慢，因为它使 AI 一个接一个地产生小回复，然后将它们全部组合成一个大回复。它也可能与某些 API 选项不兼容。

## 如何让 AI 写得更少？

这主要只是 ChatGPT 或 Claude 等模型的问题。可以应用相同的策略，但方向相反。

* 降低 `响应长度` 设置的值
* 在角色的描述中给角色一个 "言简意赅" 或 "不怎么说话" 的短语。
* 给角色一个简短的第一条消息来设定聊天的基调和期望。
* 确保 `自动继续` 已关闭。

## 如何让 AI 停止写我角色的动作，并自行推动情节？

这应该在 `作者注释` 中通过以下短语的组合来处理：

* \{\{char\}\} 的回应应该只是对 \{\{user\}\} 的行为的被动和反应性的。
* 你的下一个回应应该完全来自 \{\{char\}\} 的 POV。
* 你永远不允许为 \{\{user\}\} 规定行动或言语
