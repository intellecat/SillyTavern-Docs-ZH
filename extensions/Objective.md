# 目标

### 这是什么?

Objective 扩展允许用户为 AI 指定一个在聊天期间努力实现的目标。此目标被分解为逐步任务。任务可以分支,子任务可以自动或手动创建。这提供了创建复杂任务树的能力。列表中每个任务的完成状态将在特定间隔进行检查。

这与通过提示添加静态方向不同,因为它为 AI 添加了顺序和节奏的指令,无需用户干预。它为 AI 自主努力实现目标提供了更真实的体验。

### 前置条件

开始之前,请确保您已满足以下前置条件:

- 确保您使用的是最新版本的 SillyTavern。
- 从扩展面板(堆叠方块图标)中的"下载扩展和资源"菜单安装"Objective"扩展。

### 常见用例

您的想象力是极限,您可以给 AI 任何您希望的目标,它将规划如何实现它。您可以要求它计划如何杀死恶魔、抢劫寺庙、举办奢华派对,甚至接管世界。

![Objective 设置面板](/static/extensions/objective-panel.png)

### 配置

- 扩展在扩展菜单下的 Objective 中找到。

- 在顶部文本框中键入目标,然后单击 `自动生成任务`。这会向连接的 API 发送请求,并要求它提供与您键入的目标匹配的任务列表。

*注意:单击自动生成任务将在添加新任务之前删除当前所选目标的所有现有任务。*

- 收到 AI 的响应后,将在目标输入框下方的空间中自动创建任务列表。任务可以在创建后编辑。

- 在面板底部有两个框:`聊天中的位置`和`任务检查频率`
  - `聊天中的位置` - 这是您希望当前任务在提示的聊天部分中插入的"深度"。数字越低,AI 对任务的关注就越多。设置为 0 将使任务成为 AI 脑海中的主要内容。设置为高值将使任务在后台,允许 AI 专注于手头的对话,但设置得太高可能会导致 AI 永远"无法完成"任务。
  - `任务检查频率` - 这是您希望 AI 检查任务是否已完成的频率。如果设置为 `3`,AI 将在每第 3 条消息时被询问当前任务是否已完成。

- 目标、任务及其描述实时保存到当前聊天会话。自定义提示全局保存。

### 自定义提示
您可以自定义发送给 LLM 的提示以生成任务、检查任务完成情况和提示注入。编辑提示将为当前会话保存它们。可以保存和加载自定义提示以实现持久性。

- 单击编辑提示以打开提示编辑器窗口。您可以根据需要编辑提示。
- 要保存提示,请输入名称并单击保存提示。
- 要加载提示,请从下拉列表中选择提示。
- 要删除保存的提示,请从下拉列表中选择它并单击删除提示

**警告:任务检查发生在单独的 API 请求中。将任务检查频率设置为 1 将使您对 LLM 服务的 API 调用加倍。如果您使用付费服务,请小心使用。**

### 使用

默认情况下,Objective 扩展将自动跟踪所有任务及其各自的完成状态。

用户也可以随时手动创建、更新、删除和完成任务。


#### 当前任务选择

当前任务将始终是列出的第一个未完成任务。对任务的任何手动更新都将触发检查当前任务应该是什么。因此,如果您在一堆已完成的任务上方添加任务,它将被设置为当前任务。一旦完成,之前完成的任务将被跳过,下一个未完成的任务将被选择为"当前"。

在任务树中使用父/子任务时,任务按深度优先选择,这意味着所有子任务将首先按顺序选择,然后继续沿当前目标/任务的任务列表向下。

#### 分支任务

单击分支任务按钮将当前任务设置为目标,您可以在其中自动生成或手动创建子任务。您可以继续将任何子任务转换为目标并继续生成,尽情发挥。

将父任务标记为完成将导致扩展跳过所有子任务。当所有子任务完成时,父任务将被标记为完成

#### 手动完成任务

您可以通过`单击其旁边的复选框`手动切换任务的完成状态。这将设置要选择的下一个未完成任务。

#### 手动任务检查

如果您想手动触发 AI 检查任务完成情况,请单击额外扩展按钮(聊天输入栏右侧的`魔杖`)并选择`手动任务检查`。

![手动任务检查](/static/extensions/task-check.png)

#### 手动添加任务

当没有任务时,会显示一个 `添加任务` 按钮,允许您手动创建第一个任务。

如果已经存在其他任务,请单击任何任务右侧的 `+` 按钮以在其后插入新任务。

#### 删除任务

单击红色 `x` 删除现有任务。下一个未完成的任务将自动选择为当前任务。

删除具有子任务的任务将删除所有子任务及其后代。

#### 隐藏任务

如果您想不知道 AI 正在尝试完成哪些任务,请勾选 `隐藏任务` 框以隐藏任务列表并使 AI 的意图成为一个谜。为了 100% 的神秘性,请在单击 `自动生成任务` 之前执行此操作!
